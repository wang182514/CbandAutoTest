# C波段射频模块自动化测试系统 — 可改进点汇总

> 由 Agent 基于当前代码与产线使用场景整理，供后续迭代参考。

---

## 一、UI 排布可改进点

| 区域 | 当前问题 | 建议改进 |
|---|---|---|
| **左侧面板过长** | 仪器状态、快速设置、测试控制依次纵向堆叠；在 1080p 以下屏幕，测试按钮可能被挤到可视区域外 | 采用 `QSplitter` 左右分隔；或把「测试控制」放到上方/独立区域，确保 5 个测试按钮始终可见 |
| **仪器状态辨识度低** | 仅用文本 `✓`/`✗` 显示，颜色区分不明显 | 增加彩色状态灯（绿/红/灰 QLabel），一眼判断哪台仪表未连接 |
| **序列号输入无校验** | 任意输入均可运行，可能导致报告命名混乱 | 增加正则校验（如 `2026xxxxNxxx`），为空时提示并禁用「运行」 |
| **单个测试按钮与结果显示脱节** | 测试按钮在左侧，结果表格在右侧，无状态映射 | 按钮旁增加小型状态指示灯，或在结果表格中按测试名固定行显示 |
| **进度条位置不显眼** | 位于左侧面板底部，且测试开始后才会显示 | 进度条放到窗口底部状态栏上方，始终可见或更醒目 |
| **报告生成靠手动** | 用户容易忘记点「写入报告」 | 测试完成后自动弹出提示并默认生成报告；或提供「自动保存报告」复选框 |
| **设置对话框 Tab 过多** | 9 个 Tab，产线人员查找参数困难 | 合并为 5 个 Tab：仪器、产品/线损、RX 测试、TX 测试、报告；或增加搜索框 |
| **缺少快捷操作** | 没有「打开报告目录」「查看最近报告」入口 | 工具栏/菜单增加「打开 output/reports」「打开 output/screenshots」 |
| **日志区域信息密度低** | 多行日志混合，关键通过/失败不突出 | 对 PASS/FAIL 使用颜色高亮；增加「仅显示错误」过滤按钮 |
| **窗口默认尺寸** | 1200×800 在产线小屏电脑上可能偏大 | 默认 1100×720，并记住用户上次调整的尺寸 |

## 二、测试流程缺陷

### 2.1 仪器状态管理缺陷

- **测试开始前没有统一复位**：各测试模块假设 VSG/SA/电源处于已知状态。如果上一个测试异常退出，VSG 可能仍处于扫频模式、电源可能仍开启。
- **RX NF 异常时电源未关闭**：`rx_nf.py` 仅在正常结束时关闭 RX 电源，`except` 分支未关闭，存在硬件安全隐患。
- **TX Flatness 异常时 VSG 可能保持扫频/射频开启**：`finally` 只清 marker，未调用 `vsg.rf_off()` / `vsg.set_cw_mode()`。
- **未校验仪器模板是否存在**：直接调用 `sa.load_state()`，若仪器内无该模板，错误提示滞后。

### 2.2 配置与边界处理缺陷

- **硬编码等待时间**：
  - `tx_rx_influence.py` 中 TX 开启后固定等待 10 秒，未暴露到配置。
  - `sa_marker_noise()` 固定 3 秒等待，窄 RBW 时可能不够。
- **线损 offset 未复位**：`tx_gain.py` 每次设置 SA offset，测试结束后未归零，可能影响后续测试。
- **序列号/配置未校验**：空序列号仍可运行并生成 `测试记录_UNKNOWN.txt`。
- **缺少测试使能开关生效检查**：`default_settings.json` 中各测试有 `enabled` 字段，但代码里未读取，UI 也没有对应复选框。

### 2.3 报告与输出缺陷

- **命令行模式不生成报告**：`--headless` 只打印结果，不会写 TXT/DOCX。
- **DOCX 模板路径跳到项目根目录上级**：`main_window.py` 中 `template = os.path.join(base_dir, "..", "...")`，如果项目独立部署会找不到模板。
- **截图失败不阻塞但无明确提示**：只是日志 warning，UI 结果表格不会体现。

### 2.4 可靠性与可维护性缺陷

- **测试间耦合**：`tx_flatness_pn` 先关 RX 电源再开 TX 电源，但没有回退逻辑；如果用户只运行单个 TX 测试，RX 电源状态取决于之前运行过什么。
- **异常后状态不一致**：`TestBase` 缺少统一的 `reset_instruments()` 方法，建议在每次测试 `run()` 前后执行标准化复位。
- **QThread 停止只是设置标志位**：`request_stop()` 不会让正在阻塞的 SCPI 查询立即退出，停止响应可能延迟数十秒。

---

## 三、建议优先修复项

1. 在 `TestBase` 增加 `reset_instruments()`，每个测试 `try` 前调用。
2. 所有测试的 `finally` 增加电源关闭 + VSG 射频关闭。
3. 在 `main_window.py` 运行测试前校验序列号、仪器连接、测试使能开关。
4. 把 `tx_rx_influence` 的 10s 等待和 `sa_marker_noise` 的 3s 等待放到配置文件中。
5. `--headless` 模式运行结束后自动生成 TXT 报告。
6. UI 增加仪器状态彩色指示灯，并把「运行全部/单个测试」按钮固定在最显眼位置。
